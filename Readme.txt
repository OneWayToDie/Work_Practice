											Work_Practice — учебная практика (вариант 26)
Work_Practice — это приложение, разработанное в рамках учебной практики по направлению «Прикладная информатика в промышленности». Оно состоит из двух частей:

WPF-приложение с тремя заданиями (числа, товары, односвязный список) в едином стиле «Kaspersky».

Консольный лаунчер с анимацией кота, интерактивным решением заданий и функцией обновления.

												      Возможности

Задание 1 – последовательность целых чисел, обработка через процедуру и функцию (подсчёт цифр и минимальной цифры).
----------------------------------------------------------------------------------------------------------------------------
Задание 2 – управление базой товаров (CRUD, фильтрация, сортировка пузырьком, сохранение/загрузка XML/TXT).
----------------------------------------------------------------------------------------------------------------------------
Задание 3 – односвязный список: собственная реализация и LinkedList<T>, перенос 3-го элемента в начало.
----------------------------------------------------------------------------------------------------------------------------
Консольный лаунчер – красивое меню с иконками, анимация кота, скачивание обновлений с Google Drive, просмотром решений в консоли и возможность оставлять отзывы.

Единый стиль – тёмная тема с золотыми акцентами, кастомные диалоги, адаптация под Windows Terminal и обычную консоль.

														Технологии

WPF – основное приложение, MVVM архитектура, DataGrid для бд(на полноценную базу данных времени не хватило), кастомные стили окон.

.NET 8 – целевая платформа.

XML-сериализация – хранение данных товаров.

Адаптация лаунчера под консоль и Windows Terminal – полноцветный вывод, псевдографика, асинхронное скачивание.

											Дополнительные исправленные баги в ходе финальной доработки
1. Падение консоли при сжатии окна (IOException)
   При уменьшении окна консоли программа падала из-за вызова Console.SetCursorPosition() с координатами за пределами нового буфера. Решение: try-catch в RedrawScreen, DrawCursor, SetCursorToInput, PrintFooter. При восстановлении размера экран перерисовывается автоматически.
2. StackOverflowException при массовой защите
   Первая попытка замены всех SetCursorPosition на SafeSetCursorPosition через replaceAll нарушила структуру кода → рекурсия → StackOverflow. Решение: восстановление оригинала + try-catch только в 5 конкретных местах.
3. Сохранение WPF не в тот файл
   Данные всегда писались в products.xml даже при загрузке другого файла. Решение: currentFilePath/currentFileName в Task2ViewModel, обновление пути в ProductDataService, индикатор файла в интерфейсе.
4. Дублирование при дополнении в лаунчере
   При повторном дополнении записи дублировались (15+1=16, повторно → 31). Решение: фильтрация дубликатов по всем полям перед объединением.
5. Некорректный путь установки Inno Setup
   PrivilegesRequired=lowest переключал путь в AppData\Local\Programs. Решение: удаление директивы.
6. Отсутствие иконки WPF в проводнике
   Решение: ApplicationIcon в .csproj с rocketMonitor.ico.
7. Падение PrintFooter при сжатии
   Console.WindowHeight - 1 мог стать отрицательным. Решение: проверка h < 2 + try-catch.
8. Кодировка комментариев
   Русские комментарии отображались как ??????. Решение: пересохранение в UTF-8.

											Использование Inno Setup 6 для сборки инсталятора
Inno Setup — бесплатный инструмент для создания установщиков Windows. 
Принцип: «скрипт → exe».
Структура скрипта (.iss):
- [Setup] — имя, версия, путь установки, права
- [Languages] — языки (русский)
- [Tasks] — доп. действия (ярлыки на рабочем столе)
- [Files] — список файлов с исходными и целевыми путями
- [Icons] — ярлыки в меню «Пуск»
- [Run] — автозапуск после установки
Как связывается с проектом:
Скрипт ссылается на файлы из bin/Release. Процесс:
1. Собрать проект (dotnet build -c Release)
2. Скомпилировать инсталлятор (ISCC.exe setup.iss)
